# Strategy v3.1

Date: 2026-03-11

## Intent

`v3.1` is a bounded refinement of the `v3.0` HMM regime strategy. It keeps the same 4-state Gaussian HMM architecture and the same 15-minute regime / 1-minute execution split, but tightens regime confirmation and reduces requested risk after the weak first replay.

## What Changed From v3.0

The refinement kept the core model fixed:

- same 4-state HMM
- same 15-minute resample
- same 20-day rolling training window
- same daily retrain cadence
- same long-only execution scope

The bounded parameter changes were:

- bull entry probability: `0.62` -> `0.635`
- bull continuation probability: `0.58` -> `0.595`
- bear exit probability: `0.52` -> `0.49`
- stress exit probability: `0.48` -> `0.45`
- minimum volume ratio: `1.05` -> `1.08`
- trailing stop multiple: `0.75` -> `0.70`
- partial take-profit fraction: `0.40` -> `0.45`
- time stop: `180` -> `150` bars
- requested risk fraction: `0.0015` -> `0.0013`
- max requested notional fraction of cash: `0.08` -> `0.07`

## Why These Changes Were Made

The `v3.0` replay showed:

- too many trades
- weak expectancy
- negative realized PnL
- large drawdown relative to the baseline

The offline advisor pushed in the same direction:

- tighten regime confirmation
- require better trade quality
- cut overactivity
- reduce realized damage when the regime estimate is wrong

`v3.1` therefore focuses on:

- slightly harder bull-regime entry gating
- faster exit behavior once the bull regime decays
- lower requested risk
- more profit captured on partial exits

## Interpretation

`v3.1` was directionally better than `v3.0`, but still failed the acceptance bar:

- expectancy remains negative
- win rate remains too low
- trade count did not contract enough
- the strategy is still materially worse than the existing `v2.2` baseline

The refinement confirms that the HMM architecture is running correctly, but the current entry timing and trade-quality filters are not selective enough yet. The next revision should focus on stricter multi-timeframe confirmation and more aggressive overtrading control before changing the HMM structure itself.
