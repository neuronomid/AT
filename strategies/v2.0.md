# Strategy V2.0

## Goal

Replace the v1 minute-momentum bot with a more selective `ETH/USD` research strategy that:

- trades both long and short in backtests
- uses regime classification before entering
- defines `1R` risk explicitly with hard stop logic
- scales out at `1R` and trails the remainder toward `2R`
- limits weak, noisy, over-frequent entries
- uses an LLM only in the offline review loop

## Broker Constraint

Alpaca crypto spot should be treated as `long/flat` for live execution.

- `Long` entries and `exit` orders can be paper-executed through Alpaca.
- `Short` entries are implemented for backtests and research evaluation only.
- If the live agent emits a short entry on Alpaca crypto spot, the runtime must skip execution and log the reason.

## Timeframe

- Primary decision timeframe: `1 minute`
- Feature horizons:
  - `3 bars`
  - `5 bars`
  - `15 bars`
  - `30 bars`

## Regime Model

The strategy classifies each bar into one of four regimes:

- `uptrend`
- `downtrend`
- `range`
- `chaotic`

The first implementation uses a lightweight probabilistic regime filter built from:

- `return_15_bps`
- `return_30_bps`
- `volatility_5_bps`
- `volatility_15_bps`
- `volatility_30_bps`
- `zscore_30`

The model outputs:

- regime label
- regime probability
- directional probabilities for long and short bias

Trading rules by regime:

- `uptrend`: allow long setups only
- `downtrend`: allow short setups only
- `range`: default to no-trade
- `chaotic`: no-trade

## Entry Logic

Long entry requires:

- regime = `uptrend`
- regime probability above threshold
- short-term momentum positive
- spread below threshold
- volatility below chaos threshold
- expected edge greater than zero

Short entry requires:

- regime = `downtrend`
- regime probability above threshold
- short-term momentum negative
- spread below threshold
- volatility below chaos threshold
- expected edge greater than zero

## Risk Model

Each entry creates an explicit trade plan:

- initial stop = `1R`
- `R` is volatility-adjusted and clipped between min and max stop sizes
- position size remains cash-fraction based
- one position at a time
- hourly trade cap remains active
- cooldown remains active

Default exit plan:

- take `50%` off at `1R`
- move stop to breakeven after partial take-profit
- trail the remainder toward `2R`
- force close on time stop if the trade stalls

## Exit Logic

A position can close because of:

- hard stop loss
- partial take-profit at `1R`
- trailing stop after partial take-profit
- hard cap at `2R`
- time stop
- analyst-driven regime exit
- forced end of backtest window

## LLM Learning Loop

The LLM is not allowed to self-modify the live agent.

Allowed use:

- summarize losing trade clusters
- detect recurring mistakes by regime and setup
- propose new filters and threshold adjustments
- suggest candidate parameter updates for backtests

Required guardrail:

- every LLM suggestion must be tested through backtest and walk-forward replay before adoption

## Implementation Targets

Version `2.0` must include:

1. Regime-aware analyst with long and short signals
2. Explicit stop loss, partial TP, trailing remainder, and time stop in the backtester
3. Offline LLM strategy advisor command that writes recommendations to disk
4. Default policy definitions for the new strategy family
5. Tests covering:
   - regime classification
   - short entry
   - stop/TP behavior
   - advisor prompt generation

## Out of Scope for This Iteration

- true quote-level fill modeling
- exchange-native bracket orders for live crypto execution
- automatic LLM-driven policy promotion
- live shorting on Alpaca crypto spot
