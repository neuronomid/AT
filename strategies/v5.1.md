# V5.1 Strategy Explanation

`V5.1` is the MT5 demo-paper workflow for `BTCUSD`.

It is a separate path from:

- the Alpaca `ETH/USD` research workflow
- the older MT5 `V5.0` `EURUSD` workflow

## What V5.1 is trying to do

`V5.1` is now a `1-minute` scalping system.

The idea is:

- use `5m` to understand the nearby trend and bias
- use `1m` as the main execution timeframe
- use synthetic `20s` bars for lower-timeframe confirmation
- enter more readily when the microstructure supports the trend
- harvest profit early instead of holding for long runner targets

This is not meant to wait for a huge move to `2R+`.
It is built to take smaller, repeatable moves and get out.

## Entry workflow

`V5.1` still freezes the latest closed `1m` candle for the LLM.

The analyst gets up to a full minute to reason on that fixed packet.

If the analyst wants to enter, Python now queues the trade as soon as the analysis completes.
There is no delayed next-candle preflight step in this version.

The entry analyst now sees:

- synthetic `20s` bars for micro confirmation
- `1m` bars for execution timing
- `5m` bars for short higher-timeframe bias
- fast versus slow EMA gap
- short return windows
- ATR in basis points
- breakout distance
- close location inside the latest bar range
- body percentage of the latest bar
- tick-volume ratio
- spread
- trimmed recent reflections and lessons

The analyst can only answer:

- `enter_long`
- `enter_short`
- `hold`

It still cannot place prices or lot sizes directly.

## Philosophy change

The important change from the older `V5.1` logic is:

- do not default to `hold` just because momentum is strong
- do not treat stretched movement alone as a veto
- prefer aligned continuation or shallow-pullback entries
- hold only when the `20s` tape is choppy, disagrees with `1m`, or spread and state checks are poor

So this version is intentionally easier to enter than the old BTC runner design.

## Risk rules

`V5.1` is still model-guided but deterministic on safety.

The default risk envelope is:

- `0.20%` to `0.50%` of equity per trade
- `1.5%` daily loss kill switch
- max `15` new entries per rolling hour
- one open `BTCUSD` ticket at a time
- no scale-in stacking

## Synthetic `20s` bars

The MT5 EA still publishes native `1m`, `5m`, and `15m` bars.

Python now builds synthetic `20s` bars from the live bridge snapshots.

Those bars:

- use midpoint price for OHLC
- use snapshot count as the base activity proxy
- keep a rolling lookback of `90` closed bars
- require `30` closed bars before new entries are allowed

No fake ticks are inserted during occasional snapshot gaps.

## How stop loss is placed

The stop still uses the last `20` closed `1m` candles plus `ATR(14)`.

For longs:

- `structure_stop = lowest low of last 20 bars - 0.25 * ATR`
- `vol_stop = entry - 1.00 * ATR`
- `raw_stop = max(structure_stop, vol_stop)`

For shorts:

- `structure_stop = highest high of last 20 bars + 0.25 * ATR`
- `vol_stop = entry + 1.00 * ATR`
- `raw_stop = min(structure_stop, vol_stop)`

Then the distance from entry to stop is clamped to:

- minimum previous-candle shadow size
- minimum `0.80 * ATR`
- maximum `2.20 * ATR`

After that, the order is rounded to the broker tick size and stop-distance rules.

## How take profit works

`V5.1` now uses a scalp ladder instead of a runner target.

The planned reward ladder is:

- soft `TP0.5 = 0.5R`
- hard `TP1 = 1.0R`

So the runtime takes a partial at `0.5R`, then leaves the remaining half to either hit the hard `1.0R` target or the trailed stop.

## Open-trade management

Normal profit taking is now deterministic.

It is checked on every fresh snapshot.

The deterministic scalp rules are:

- do not scalp out on the entry candle; the position must survive at least `2` closed `1m` candles
- at `>= 0.5R`, close `50%` of the original size and move the stop on the remainder to breakeven
- after that partial, the remaining half is left with broker-side stop and `TP1`

There is no manager LLM in this version.
Once the partial is taken, the remaining half is meant to finish by either hitting breakeven or `TP1`.

## Feedback loop

When a trade closes, `V5.1` still stores:

- the reflection
- realized `R`
- exit reason
- lesson candidates

Those lessons are fed back into later entry decisions, but only from the latest `4` trades.
